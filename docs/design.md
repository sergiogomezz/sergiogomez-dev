# Design Direction

## Product Concept

The website should work as an interactive CV for an AI / ML Engineer. The PDF remains the formal document, while the website should make the same factual profile easier to scan, navigate, and understand.

## Initial Direction

- Professional, technical, and restrained.
- Clear information hierarchy over decorative effects.
- Strong readability on desktop and mobile.
- Case-study friendly structure for AI, ML, data, and agent projects.
- Warm enough to feel personal, but not informal or gimmicky.

## Layout Principles

- Start with the actual portfolio content, not a marketing-style landing page.
- Make the first viewport communicate name, role, location, current focus, and CV/contact links.
- Use sections that map to visitor intent:
  - quick profile scan,
  - experience,
  - projects,
  - education,
  - skills,
  - contact.
- Keep navigation simple and predictable.
- Use progressive detail: summaries first, deeper project or experience details later.

## Visual Principles

- Do not commit to a final color palette yet.
- Avoid generic "futuristic AI" visuals unless they support the content.
- Avoid a one-note palette dominated by a single hue.
- Prefer typography, spacing, structure, and subtle technical details over heavy decoration.
- Use cards only for repeated items or clearly framed content, not as nested page sections.
- Keep buttons and links visually distinct and accessible.

## Accessibility And Responsiveness

- Maintain semantic HTML structure.
- Use accessible link names, especially for the CV PDF.
- Preserve sufficient color contrast.
- Ensure keyboard navigation works for interactive elements.
- Avoid text overlap and layout shifts across mobile and desktop.
- Do not rely on animation to communicate essential information.
